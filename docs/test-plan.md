## Affected Pages/Routes
- `/` (leaderboard) — renders robustness scores from a committed results JSON; must load fast and correctly with 0, 1, and many models.
- `/playground` — pick/paste a page, toggle a hidden injection, run ONE live attack; verify clean vs poisoned output side by side, the canary leak indicator, and a clear error (not a hang) on model timeout.
- `api/attack` (serverless) — single live attack; must stay under the 60s Vercel budget and rate-limit per IP.

## Key Interactions to Verify
- Toggle an injection technique on the playground and confirm the AI's output changes (or is blocked when defense is ON).
- Defense ON vs OFF toggle reflects in the result and the canary indicator.
- Leaderboard sort/filter by model and attack technique.

## Edge Cases
- Attack payload that itself contains the `</untrusted>` delimiter (must be escaped, not allowed to break out). SECURITY-CRITICAL.
- The judge model fed an injection in the content it is grading (judge must not obey it). SECURITY-CRITICAL.
- Unicode-smuggled instruction (zero-width / tag chars) — sanitizer must strip before the model sees it.
- Page fetch fails / returns non-HTML / is huge — clear error, no hang.
- Model API timeout or rate-limit — clear user-facing error.
- Empty corpus / malformed payload JSON — schema validation rejects it.

## Critical Paths (must work end to end)
- Playground: fetch page -> build poisoned content -> call model -> judge verdict -> render clean vs poisoned. [E2E]
- Leaderboard: committed results JSON -> render ranked table. [E2E]

## Evals (the heart of the project)
- **Judge accuracy:** judge verdicts vs a hand-labeled set of >=50 samples, stratified across attack techniques and models. Report accuracy.
- **Defense before/after:** attack-success rate with the defense pipeline ON vs OFF, across the corpus x models. This is the headline number ("X% -> Y%").

## Unit coverage required
- `defense/sanitize.ts`: NFKC, zero-width strip, tag-char strip, HTML-comment + hidden-CSS strip, delimiter escape.
- `defense/canary.ts`: leak detected when secret present; not detected on clean output.
- `judge/judge.ts`: canary path, rule path, LLM path; judge resists injection.
- `corpus/loader.ts`: accept valid payloads, reject malformed.
- `harness/runSuite.ts`: buildPoisonedContent() per technique; aggregates to a per-model score.
